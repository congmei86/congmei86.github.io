1 标题

从 h2 到 h6 的标题在每个级别上都加上一个 ＃:

1## h2 标题
2### h3 标题
3#### h4 标题
4##### h5 标题
5###### h6 标题

输出的 HTML 看起来像这样:

1<h2>h2 标题</h2>
2<h3>h3 标题</h3>
3<h4>h4 标题</h4>
4<h5>h5 标题</h5>
5<h6>h6 标题</h6>
标题 ID

要添加自定义标题 ID, 请在与标题相同的行中将自定义 ID 放在花括号中:

1### 一个很棒的标题 {#custom-id}

输出的 HTML 看起来像这样:

1<h3 id="custom-id">一个很棒的标题</h3>

4 段落

按照纯文本的方式书写段落, 纯文本在呈现的 HTML 中将用 <p>/</p> 标签包裹.

如下段落:

1Lorem ipsum dolor sit amet, graecis denique ei vel, at duo primis mandamus. Et legere ocurreret pri,
2animal tacimates complectitur ad cum. Cu eum inermis inimicus efficiendi. Labore officiis his ex,
3soluta officiis concludaturque ei qui, vide sensibus vim ad.

输出的 HTML 看起来像这样:

1<p>Lorem ipsum dolor sit amet, graecis denique ei vel, at duo primis mandamus. Et legere ocurreret pri, animal tacimates complectitur ad cum. Cu eum inermis inimicus efficiendi. Labore officiis his ex, soluta officiis concludaturque ei qui, vide sensibus vim ad.</p>

可以使用一个空白行进行换行.

6 强调

加粗

用于强调带有较粗字体的文本片段.

以下文本片段会被 渲染为粗体.

1**渲染为粗体**
2__渲染为粗体__

输出的 HTML 看起来像这样:

1<strong>渲染为粗体</strong>

斜体

用于强调带有斜体的文本片段.

以下文本片段被 渲染为斜体.

1*渲染为斜体*
2_渲染为斜体_

输出的 HTML 看起来像这样:

1<em>渲染为斜体</em>

删除线

按照 GFMGitHub flavored Markdown 你可以使用删除线.

1~~这段文本带有删除线.~~

呈现的输出效果如下:

这段文本带有删除线.

输出的 HTML 看起来像这样:

1<del>这段文本带有删除线.</del>

组合

加粗, 斜体, 和删除线可以 组合使用.

1***加粗和斜体***
2~~**删除线和加粗**~~
3~~*删除线和斜体*~~
4~~***加粗, 斜体和删除线***~~

呈现的输出效果如下:

加粗和斜体

删除线和加粗

删除线和斜体

加粗, 斜体和删除线

输出的 HTML 看起来像这样:

1<em><strong>加粗和斜体</strong></em>
2<del><strong>删除线和加粗</strong></del>
3<del><em>删除线和斜体</em></del>
4<del><em><strong>加粗, 斜体和删除线</strong></em></del>

7 引用

用于在文档中引用其他来源的内容块.

在要引用的任何文本之前添加 >:

1> **Fusion Drive** combines a hard drive with a flash storage (solid-state drive) and presents it as a single logical volume with the space of both drives combined.

呈现的输出效果如下:

Fusion Drive combines a hard drive with a flash storage (solid-state drive) and presents it as a single logical volume with the space of both drives combined.

输出的 HTML 看起来像这样:

1<blockquote>
2  <p>
3    <strong>Fusion Drive</strong> combines a hard drive with a flash storage (solid-state drive) and presents it as a single logical volume with the space of both drives combined.
4  </p>
5</blockquote>

引用也可以嵌套:

1> Donec massa lacus, ultricies a ullamcorper in, fermentum sed augue.
2Nunc augue augue, aliquam non hendrerit ac, commodo vel nisi.
3>> Sed adipiscing elit vitae augue consectetur a gravida nunc vehicula. Donec auctor
4odio non est accumsan facilisis. Aliquam id turpis in dolor tincidunt mollis ac eu diam.

呈现的输出效果如下:

Donec massa lacus, ultricies a ullamcorper in, fermentum sed augue. Nunc augue augue, aliquam non hendrerit ac, commodo vel nisi.

Sed adipiscing elit vitae augue consectetur a gravida nunc vehicula. Donec auctor odio non est accumsan facilisis. Aliquam id turpis in dolor tincidunt mollis ac eu diam.

任务列表

任务列表使你可以创建带有复选框的列表. 要创建任务列表, 请在任务列表项之前添加破折号 (-) 和带有空格的方括号 ([ ]). 要选择一个复选框，请在方括号之间添加 x ([x]).

1- [x] Write the press release
2- [ ] Update the website
3- [ ] Contact the media

呈现的输出效果如下:

  • Write the press release
  • Update the website
  • Contact the media

行内代码

用 ` 包装行内代码段.

1在这个例子中, `<section></section>` 会被包裹成 **代码**.

呈现的输出效果如下:

在这个例子中, <section></section> 会被包裹成 代码.

输出的 HTML 看起来像这样:

1<p>
2  在这个例子中, <code>&lt;section&gt;&lt;/section&gt;</code> 会被包裹成 <strong>代码</strong>.

缩进代码

将几行代码缩进至少四个空格，例如:

1    // Some comments
2    line 1 of code
3    line 2 of code
4    line 3 of code

呈现的输出效果如下:

// Some comments
line 1 of code
line 2 of code
line 3 of code

输出的 HTML 看起来像这样:

1<pre>
2  <code>
3    // Some comments
4    line 1 of code
5    line 2 of code
6    line 3 of code
7  </code>
8</pre>

围栏代码块

使用 “围栏” ``` 来生成一段带有语言属性的代码块.

1```markdown
2Sample text here...
3```

输出的 HTML 看起来像这样:

1<pre language-html>
2  <code>Sample text here...</code>
3</pre>

10 表格

通过在每个单元格之间添加竖线作为分隔线, 并在标题下添加一行破折号 (也由竖线分隔) 来创建表格. 注意, 竖线不需要垂直对齐.

1| Option | Description |
2| ------ | ----------- |
3| data   | path to data files to supply the data that will be passed into templates. |
4| engine | engine to be used for processing templates. Handlebars is the default. |
5| ext    | extension to be used for dest files. |

呈现的输出效果如下:

Option Description
data path to data files to supply the data that will be passed into templates.
engine engine to be used for processing templates. Handlebars is the default.
ext extension to be used for dest files.

输出的 HTML 看起来像这样:

 2  <thead>

 4      <th>Option</th>
 5      <th>Description</th>
 6    </tr>
 7  </thead>

10      <td>data</td>
11      <td>path to data files to supply the data that will be passed into templates.</td>
12    </tr>
13    <tr>
14      <td>engine</td>
15      <td>engine to be used for processing templates. Handlebars is the default.</td>
16    </tr>
17    <tr>

定位标记

定位标记使你可以跳至同一页面上的指定锚点. 例如, 每个章节:

1## Table of Contents
2  * [Chapter 1](#chapter-1)
3  * [Chapter 2](#chapter-2)
4  * [Chapter 3](#chapter-3)

将跳转到这些部分:

1## Chapter 1 <a id="chapter-1"></a>
2Content for chapter one.
3
4## Chapter 2 <a id="chapter-2"></a>
5Content for chapter one.
6
7## Chapter 3 <a id="chapter-3"></a>
8Content for chapter one.
Note
定位标记的位置几乎是任意的. 因为它们并不引人注目, 所以它们通常被放在同一行了.

12 脚注

脚注使你可以添加注释和参考, 而不会使文档正文混乱. 当你创建脚注时, 会在添加脚注引用的位置出现带有链接的上标编号. 读者可以单击链接以跳至页面底部的脚注内容.

要创建脚注引用, 请在方括号中添加插入符号和标识符 ([^1]). 标识符可以是数字或单词, 但不能包含空格或制表符. 标识符仅将脚注引用与脚注本身相关联 - 在脚注输出中, 脚注按顺序编号.

在中括号内使用插入符号和数字以及用冒号和文本来添加脚注内容 ([^1]：这是一段脚注). 你不一定要在文档末尾添加脚注. 可以将它们放在除列表, 引用和表格等元素之外的任何位置.

1这是一个数字脚注[^1].
2这是一个带标签的脚注[^label]
3
4[^1]: 这是一个数字脚注
5[^label]: 这是一个带标签的脚注

这是一个数字脚注1.

这是一个带标签的脚注2

13 图片

1![Alt text][id]

1[id]: https://octodex.github.com/images/dojocat.jpg  "The Dojocat"
